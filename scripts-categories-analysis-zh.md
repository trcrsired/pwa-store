## scripts/generate_wrappers.py 关键字段说明

下面这部分是基于 `scripts/generate_wrappers.py` 中 `WRAPPERS = [...]` 的**实际条目结构**重新整理的。

严格来说，单个 wrapper 配置对象真正支持并被脚本读取的关键字段只有 5 个：

- `name`
- `url`
- `keyify_name`
- `apptype`
- `icon`

其中：

- **必填字段**：`name`、`url`
- **可选字段**：`keyify_name`、`apptype`、`icon`

---

### 1. 关键字段总表

| 字段名 | 是否必填 | 类型 | 含义 | 示例 | 作用 | 默认行为 |
|---|---|---|---|---|---|---|
| `name` | 是 | `string` | 应用或站点显示名称 | `"name": "iCloud"` | 用于页面标题、manifest 名称，也可作为默认 key 的生成来源 | 无默认值，缺失会报错 |
| `url` | 是 | `string` | 目标网站地址 | `"url": "https://www.icloud.com"` | 写入 `config.js`，作为包装目标网址 | 无默认值，缺失会报错 |
| `keyify_name` | 否 | `string` | 手动指定目录 key / 标识名 | `"keyify_name": "icloud"` | 决定输出目录名、manifest 路径、默认图标名 | 未填写时自动由 `name` 生成 |
| `apptype` | 否 | `string` | 指定应用分类 | `"apptype": "msedge"` | 决定生成到哪个目录、使用哪个图标目录 | 默认是 `"wrapper"` |
| `icon` | 否 | `string` | 手动指定图标文件名 | `"icon": "apple.webp"` | 指定复制哪个图标文件 | 默认使用 `${key}.webp` |

---

### 2. 字段逐项说明

#### `name`

| 项目 | 说明 |
|---|---|
| 字段名 | `name` |
| 含义 | 站点或应用的显示名称 |
| 是否必填 | 是 |
| 示例 | `{ "name": "GoFundMe", "url": "https://www.gofundme.com" }` |
| 脚本中的使用位置 | `name = wrapper["name"]` |
| 作用 | 1. 生成 `index.html` 中的标题；2. 写入 `manifest.json` 的 `name` 与 `short_name`；3. 未提供 `keyify_name` 时参与 key 自动生成 |
| 注意事项 | 名称可带空格、大小写、中文；但若依赖自动 keyify，最终目录 key 会被转小写并移除空格与 `.` |

#### `url`

| 项目 | 说明 |
|---|---|
| 字段名 | `url` |
| 含义 | wrapper 指向的目标网址 |
| 是否必填 | 是 |
| 示例 | `{ "name": "WBTV", "url": "https://www.wbtv.com" }` |
| 脚本中的使用位置 | `url = wrapper["url"]` |
| 作用 | 写入生成出的 `config.js` 中，供前端包装逻辑读取 |
| 注意事项 | 脚本不会校验 URL 是否可访问，只会原样写入 |

生成效果示例：

```js
window.appConfig = {
  title: "WBTV",
  url: "https://www.wbtv.com"
};
```

#### `keyify_name`

| 项目 | 说明 |
|---|---|
| 字段名 | `keyify_name` |
| 含义 | 手动指定该应用的目录 key / 唯一标识 |
| 是否必填 | 否 |
| 示例 | `{ "name": "Novaya Gazeta Europe", "url": "https://novayagazeta.eu", "keyify_name": "novayagazeta" }` |
| 脚本中的使用位置 | `key = wrapper.get("keyify_name", keyify(name))` |
| 作用 | 1. 决定输出目录名；2. 决定 manifest 中的 `id`、`start_url`、`scope`；3. 决定默认图标名 |
| 默认行为 | 若未填写，则使用 `keyify(name)` 自动生成 |
| 注意事项 | 对带空格、特殊字符、较长名称的站点，推荐显式填写，避免目录名不稳定 |

自动生成规则：

```python
def keyify(name):
    return name.lower().replace(".", "").replace(" ", "")
```

也就是：

- 转小写
- 去掉英文句号 `.`
- 去掉空格

示例：

| 原始 `name` | 自动生成的 key |
|---|---|
| `Live Science` | `livescience` |
| `Base64.Guru` | `base64guru` |
| `GoFundMe` | `gofundme` |

#### `apptype`

| 项目 | 说明 |
|---|---|
| 字段名 | `apptype` |
| 含义 | 指定该条目属于哪种应用分类 |
| 是否必填 | 否 |
| 示例 | `{ "name": "iCloud", "url": "https://www.icloud.com", "keyify_name": "icloud", "apptype": "msedge" }` |
| 脚本中的使用位置 | `apptype = wrapper.get("apptype", "wrapper")` |
| 作用 | 1. 决定输出目录；2. 决定读取哪个图标目录；3. 决定 manifest 路径前缀 |
| 默认行为 | 默认为 `"wrapper"` |
| 可选值 | `wrapper`、`extension`、`msedge` |
| 注意事项 | 如果填了脚本不支持的值，脚本在读取映射表时会报错 |

`apptype` 对应关系：

| `apptype` 值 | 输出目录 | 图标目录 | manifest 路径前缀 |
|---|---|---|---|
| `wrapper` | `docs/wrappers/` | `docs/store/icons/wrappers/` | `/wrappers/` |
| `extension` | `docs/extensions/` | `docs/store/icons/extensions/` | `/extensions/` |
| `msedge` | `docs/msedge/` | `docs/store/icons/msedge/` | `/msedge/` |

#### `icon`

| 项目 | 说明 |
|---|---|
| 字段名 | `icon` |
| 含义 | 手动指定使用哪个图标文件 |
| 是否必填 | 否 |
| 示例 | `{ "name": "Apple Store", "url": "https://www.apple.com/store", "keyify_name": "applestore", "icon": "apple.webp" }` |
| 脚本中的使用位置 | `icon_file = wrapper.get("icon", f"{key}.webp")` |
| 作用 | 从对应图标目录复制指定文件到目标目录下的 `icons/icon.webp` |
| 默认行为 | 如果不写，则默认查找 `${key}.webp` |
| 注意事项 | 若图标文件不存在，脚本不会终止，但会输出 `Warning: icon not found` |

默认图标规则示例：

| `keyify_name` | 默认图标文件 |
|---|---|
| `icloud` | `icloud.webp` |
| `gofundme` | `gofundme.webp` |
| `novayagazeta` | `novayagazeta.webp` |

---

### 3. 字段之间的关系

| 关系 | 说明 | 示例 |
|---|---|---|
| `name -> keyify_name` | 未填写 `keyify_name` 时，脚本会根据 `name` 自动生成 | `Live Science -> livescience` |
| `keyify_name -> icon` | 未填写 `icon` 时，默认图标文件名为 `${key}.webp` | `gofundme -> gofundme.webp` |
| `apptype -> 输出目录` | `apptype` 决定生成到 `wrappers`、`extensions` 或 `msedge` | `msedge -> docs/msedge/...` |
| `apptype + keyify_name -> manifest 路径` | 决定 `id`、`start_url`、`scope` | `/msedge/icloud/` |
| `name + url -> config.js` | 生成页面标题和目标地址配置 | `title: "iCloud", url: "https://www.icloud.com"` |

### 5. 脚本级别的补充关键配置

下面这些不是 `WRAPPERS` 单条对象里的字段，但属于 `generate_wrappers.py` 的关键配置：

| `CONFIG_JS` | `str` | `config.js` 模板 | 生成目标站点配置 |
| `INDEX_HTML` | `str` | `index.html` 模板 | 生成包装页面 |
| `build_manifest()` | `function` | 生成 manifest | 让生成结果具备 PWA 安装信息 |
| `keyify()` | `function` | 自动生成 key | 用于默认目录名和图标名 |
